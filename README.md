# 수능형 수학 문항 편집기

수능·평가원 수학 문항의 지면 양식을 참고하여, 웹 브라우저에서 수학 문제를 작성하고 미리보기·이미지 저장·LaTeX 원문 내보내기까지 할 수 있는 React 기반 웹앱입니다.

이 프로젝트는 수학 문제를 빠르게 조립하기 위한 **문항 전용 편집기**입니다. 일반적인 문서 편집기처럼 모든 것을 자유 배치하는 방식이 아니라, 실제 시험지 문항처럼 보이도록 **발문, 수식 블록, 조건 박스, 보기 박스, 오지선다**를 블록 단위로 구성합니다.

## 주요 기능

### 1. 수능형 문항 미리보기

흰 배경, 검은 글씨, 문항 번호, 조건 박스, 수식 블록, 오지선다 배치를 포함한 시험지형 화면을 제공합니다. 기본 예시는 삼차함수와 조각함수 조건을 포함한 수학 문항으로 시작합니다.

### 2. 블록 기반 문항 편집

문항은 다음 블록들로 구성됩니다.

- **발문**: 문항의 시작 문장
- **문장 블록**: 발문 중간이나 결론 문장을 추가하는 일반 텍스트 블록
- **수식 블록**: 조각함수, 극한식, 적분식 등 중앙 배치가 필요한 수식 블록
- **조건 박스**: (가), (나), (다) 조건을 넣는 박스
- **보기 박스**: `<보 기>` 제목이 붙은 ㄱ, ㄴ, ㄷ 판단형 박스
- **오지선다**: ①~⑤ 선지 입력 영역

각 블록은 위아래 이동, 삭제, 정렬, 위쪽 여백, 아래쪽 여백, 글자 크기 조절이 가능합니다.

### 3. 수식 입력 보조

편집 창에는 수식·기호 삽입 탭이 있습니다.

현재 제공되는 탭은 다음과 같습니다.

- 기본
- 함수·극한
- 미분·적분
- 수열
- 삼각
- 기하·벡터
- 관계·집합
- 수능 문장

일반 문장 영역에서 수학 기호를 넣으면 자동으로 `$...$`로 감싸집니다. 이미 `$...$` 안에 커서가 있거나 수식 블록을 편집 중이면 다시 감싸지 않고 그대로 삽입됩니다.

예를 들어 문장 영역에 `함수 `를 입력한 뒤 `f(x)` 버튼을 누르면 다음처럼 들어갑니다.

```text
함수 $f(x)$
```

반면 수식 블록에서는 `f(x)`가 그대로 들어갑니다.

### 4. 오지선다 자동 배치

오지선다 선지는 길이에 따라 자동으로 배치가 바뀝니다.

- 짧은 선지: 한 줄 5개
- 중간 길이 선지: 3열
- 긴 선지: 2열
- 매우 긴 선지: 1열

띄어쓰기 없는 긴 문자열도 문항 영역 밖으로 빠져나가지 않도록 줄바꿈 처리가 되어 있습니다.

### 5. 이미지 저장

`이미지 저장` 버튼을 누르면 현재 문항 영역만 PNG 이미지로 저장됩니다. 상단 메뉴나 편집 팝업은 이미지에 포함되지 않습니다.

### 6. LaTeX 원문 보기 및 txt 저장

`LaTeX 보기` 버튼을 누르면 현재 문항의 원문을 확인할 수 있습니다. 브라우저 환경에 따라 클립보드 복사가 제한될 수 있으므로, 별도 창에서 직접 복사하거나 txt 파일로 저장할 수 있도록 구성했습니다.

### 7. 모바일·태블릿 대응

편집 화면은 팝업 형태로 열리므로 휴대폰과 태블릿에서도 문항 미리보기와 편집 영역이 겹치지 않습니다. 좁은 화면에서는 사용법, 편집, LaTeX 보기, 이미지 저장 버튼 중심으로 표시됩니다.

## 설치 및 실행

### 요구 사항

- Node.js 20 이상 권장
- npm

### 로컬 실행

```bash
npm install
npm run dev
```

개발 서버가 실행되면 브라우저에서 표시되는 주소로 접속합니다. 보통 다음과 같습니다.

```text
http://localhost:5173
```

같은 Wi-Fi에 연결된 휴대폰에서 테스트하려면 다음처럼 실행합니다.

```bash
npm run dev -- --host 0.0.0.0
```

터미널에 표시되는 `Network` 주소를 휴대폰 브라우저에서 열면 됩니다.

### 빌드

```bash
npm run build
```

빌드 결과물은 `dist` 폴더에 생성됩니다.

### 빌드 결과 미리보기

```bash
npm run preview
```

## GitHub Pages 배포

이 저장소에는 GitHub Pages 배포용 워크플로가 포함되어 있습니다.

```text
.github/workflows/deploy.yml
```

`main` 브랜치에 push가 발생하면 다음 순서로 자동 배포됩니다.

1. 저장소 체크아웃
2. Node.js 20 설정
3. 의존성 설치
4. Vite 빌드
5. `dist` 폴더 업로드
6. GitHub Pages에 배포

### GitHub Pages 설정 확인

저장소에서 다음 메뉴를 확인합니다.

```text
Settings → Pages → Build and deployment → Source
```

`GitHub Actions`로 설정되어 있어야 합니다.

## 사용 방법

### 1. 문항 번호 수정

`편집` 버튼을 누른 뒤 문항 번호 입력칸에서 번호를 바꿀 수 있습니다.

### 2. 발문 수정

발문 입력칸에 문항의 시작 문장을 입력합니다. `$f(x)$`, `$t$`처럼 달러 기호로 감싸면 수식으로 렌더링됩니다.

### 3. 블록 추가

편집 창의 `문항 안에 들어가는 블록` 영역에서 다음 버튼을 사용할 수 있습니다.

- `문장`: 일반 문장 추가
- `수식`: 수식 블록 추가
- `박스`: 조건 박스 추가
- `보기`: `<보 기>` 박스 추가

### 4. 블록 이동 및 삭제

각 블록 오른쪽 위의 화살표 버튼으로 순서를 바꾸고, 휴지통 버튼으로 삭제할 수 있습니다.

### 5. 정렬과 여백 조정

각 블록 아래쪽에서 다음 값을 조정할 수 있습니다.

- 왼쪽 정렬
- 중앙 정렬
- 위 공백
- 아래 공백
- 글자 크기

조건 박스는 추가로 박스 너비도 조정할 수 있습니다.

### 6. 오지선다 사용

오지선다 토글을 켜면 ①~⑤ 선지를 입력할 수 있습니다. 선지 길이에 따라 자동으로 5열, 3열, 2열, 1열 배치가 적용됩니다.

### 7. 이미지 저장

`이미지 저장` 버튼을 누르면 문항 미리보기 영역만 PNG로 저장됩니다.

### 8. LaTeX 원문 저장

`LaTeX 보기` 버튼을 누르면 현재 문항 원문이 열립니다. 내용을 직접 복사하거나 txt 파일로 저장할 수 있습니다.

## 설계 의도

이 편집기는 일반적인 WYSIWYG 문서 편집기가 아니라, 수능형 문항 제작에 필요한 구조를 빠르게 조립하는 도구입니다. 특히 다음과 같은 상황에 맞게 설계되었습니다.

- 수학 문항 초안을 빠르게 만들 때
- 조건 박스와 보기 박스가 포함된 문제를 만들 때
- 수식이 포함된 발문을 정돈된 형태로 배치하고 싶을 때
- 문제 이미지를 저장해 공유하고 싶을 때
- LaTeX 원문을 따로 보관하고 싶을 때

---

## 알려진 제한 사항

- 현재 문항 데이터는 브라우저 새로고침 후 자동 저장되지 않습니다.
- 완전한 워드프로세서처럼 자유 배치하는 기능은 제공하지 않습니다.
- 이미지 저장은 브라우저의 캔버스 렌더링 정책과 폰트 로딩 상태에 영향을 받을 수 있습니다.
- GitHub Pages에서 처음 접속할 때 빌드/배포 완료까지 시간이 필요할 수 있습니다.
